Committed to accessibility

1. Introduction

This website is partially compliant with the accessibility requirements established in the applicable regulations. The evaluation was carried out using a combined methodology of automatic auditing and manual review, using tools such as Lighthouse, TAW and various manual tests for keyboard navigation and structural content review.

The audit was conducted with reference to the criteria set out in UNE-EN 301 549, based on WCAG 2.1 Level AA, and was completed on the date corresponding to the latest accessibility review of the website. During the analysis, several aspects were identified that meet the established requirements, as well as others that require improvement, particularly regarding color contrast, keyboard accessibility and certain non-text elements.

2. Conformance

The level of conformance indicates the extent to which this website meets the established accessibility standards to ensure universal access to information and digital services. Below is the information corresponding to the latest accessibility review of the website:

  • Date: 09-12-2025
  • Guidelines applied: UNE-EN 301 549 standard based on WCAG
  • Version: WCAG 2.1
  • Audited website: https://vally.lvis.es
  • Evaluated compliance level: AA

3. Scope of the statement

This accessibility statement applies to the entire audited website and includes all pages and sections that are part of the evaluated domain. The analysis covered both the main content and navigation elements, headings, forms, images, videos, interactive components and any other functionality available to users.

The scope includes all public areas of the website reviewed during the audit, with the aim of ensuring transparency regarding the evaluated content and the level of compliance achieved. Any section, resource or functionality outside the scope will be specified in later sections of this statement.

4. Sample pages

The index page has been audited, as well as all pages accessible from the navigation menu.

5. Technologies used

The audited website includes the following technologies across its pages: html, javascript and css

6. Tools used

To evaluate the accessibility of the website, a combination of automatic tools and manual review techniques was used. These tools made it possible to identify issues related to contrast, semantic structure, keyboard navigation and the correct implementation of ARIA attributes, among other fundamental aspects.

The tools used during the audit were as follows:

  • Google Lighthouse (automatic audit integrated in Chrome DevTools)
  • TAW / TAWDis (WCAG-based accessibility validator)
  • Accessibility-focused browser extensions, including contrast checkers and analysis of ARIA roles and attributes
  • Manual keyboard-only navigation tests
  • Manual review of semantic structure (headings, lists, links and forms)
  • Verification of captions and transcripts in multimedia content

7. User agents used

The tests were carried out using Google Chrome latest

8. Error report

Below are the detailed results of the accessibility audit, including the criteria that were met, those that presented issues and those that are not applicable to the website content. This report compiles information from the different tools used and from the manual tests performed, in order to provide a clear and transparent view of the current accessibility status of the website.

Audit results

1. Criteria met:

Numerous requirements related to the correct use of ARIA roles and attributes, the presence and validity of the lang attribute, the existence of appropriate titles in documents and iframes, the correct structure of lists and headings, the presence of associated labels in forms, and the existence of discernible names for links, among others, have been successfully met.
Videos also include captions and audio transcripts.

2. Criteria with detected issues:

During the audit, issues were identified in:

  • Insufficient contrast ratio between text and background in certain elements.
  • Non-text content lacking appropriate descriptions in some cases, as indicated by TAWDis.
  • Information and relationships not fully exposed correctly to assistive technologies.
  • Keyboard navigation, with certain elements or controls not accessible without using a mouse.

3. Non-applicable criteria:

Some accessibility requirements do not apply to the website due to the absence of specific functionalities (for example, absence of downloadable documents, absence of captchas, or lack of advanced interactive modules requiring additional validation).
